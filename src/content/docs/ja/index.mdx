SnapVM は、ファイルシステム、パッケージ、設定をハイバネーション後も保持する隔離された開発ワークスペースを提供します。開発者はローカルのインフラを用意せずに、ブラウザから Linux 環境、ターミナル、エディタ、HTTP サービスへアクセスできます。

組織、ユーザー、アクセス権は SnapVM の Web コンソールで管理します。マシンはアイドル時にハイバネーションし、再接続時に復帰できるため、チームは状態を保ったままコンピュートの稼働時間を抑えられます。

SnapVM にはターミナル向けの CLI も含まれます。ブラウザ中心のマシン管理には Web コンソールを使い、ローカルターミナルからサインイン、マシンの作成や一覧表示、シェル起動、コマンド実行、チェックポイント操作を行う場合は CLI を使います。

## SnapVM が提供するもの

SnapVM は、永続ワークスペースとハイバネーションを、チームで扱いやすい Web コンソール、CLI、認証・認可、料金プラン、接続体験として提供します。

- コンソールまたは CLI からマシンを作成し、状態、ランタイム、アプリケーション URL を管理できます。
- 組織、ユーザー、アクセス権を SnapVM 側で管理し、チームごとの利用範囲を制御できます。
- ファイル、パッケージ、リポジトリ、サービス定義、設定は停止やハイバネーション後も保持されます。
- ターミナルや HTTP サービスはブラウザから開けます。CLI を使うと、ローカルターミナルからマシン内でコマンドを実行できます。

## SnapVM が違うところ

SnapVM のマシンは、短命な実行環境ではなく、必要なときに起動できる状態付きの Linux 環境として扱えます。常時稼働させなくても、作業に必要な状態を保持できるように設計されています。

## ユースケース

SnapVM は、状態を保ったまま安全にコードを動かしたい場面に向いています。代表的な用途は次のとおりです。

- AI が生成したコードを隔離された環境で実行する
- ユーザー投稿コードや信頼できないコードを安全に試す
- セッションをまたいで状態が残る開発環境を提供する
- アイドル時に停止し、リクエストで復帰する HTTP サービスを動かす
- Git リポジトリや依存関係を含むプロジェクト作業を実行する

## コア概念

### 永続性

マシンはファイルシステムを保持します。実行中に書き込んだファイル、インストールしたパッケージ、チェックアウトしたリポジトリは、アイドル後の復帰時にも利用できます。

### 自動アイドル動作

使われていないマシンはアイドル状態になり、必要に応じてハイバネーションします。次の接続やリクエストで復帰するため、常時稼働させずに状態を残せます。

### HTTP アクセス

マシン上で起動した HTTP サービスには、デプロイで提供されている場合にアプリケーション URL 経由でアクセスできます。
