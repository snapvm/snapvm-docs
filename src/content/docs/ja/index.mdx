SnapVM は [Sprites](https://docs.sprites.dev/) を基盤に、隔離された開発ワークスペースを提供します。Sprites はファイルシステム、パッケージ、設定をハイバネーション後も保持する永続クラウドコンテナです。開発者はローカルのインフラを用意せずに、ブラウザから Linux 環境、ターミナル、エディタ、HTTP サービスへアクセスできます。

組織、ユーザー、アクセス権は SnapVM の Web コンソールで管理します。マシンはアイドル時にハイバネーションし、再接続時に復帰できるため、チームは状態を保ったままコンピュートの稼働時間を抑えられます。

SnapVM は現在、Web コンソールと HTTP API を中心に提供されています。スタンドアロンの SnapVM CLI や MCP サーバーは提供していないため、このドキュメントではそれらの手順は扱いません。

## SnapVM が提供するもの

SnapVM は、Sprites をバックエンドとして利用しやすくするためのプロダクトレイヤーです。永続ワークスペースやハイバネーションといった Sprites の性質を、チームで扱いやすい Web コンソール、認証・認可、料金プラン、接続体験として提供します。

<ProductLayerDiagram locale="ja" />

- コンソールからマシンを作成し、状態、リージョン、ランタイム、ゲートウェイ URL を管理できます。
- 組織、ユーザー、アクセス権を SnapVM 側で管理し、チームごとの利用範囲を制御できます。
- Sprites の永続性とアイドル時のハイバネーションを、マシン数ベースのシンプルで予測可能なプランで扱えます。
- ターミナルや HTTP サービスへのアクセスは、ブラウザと connect gateway 経由で利用できます。

## SnapVM が違うところ

SnapVM のマシンは [Sprites](https://docs.sprites.dev/#what-makes-sprites-different) の性質を引き継ぎ、短命な実行環境ではなく、必要なときに起動できる状態付きの Linux 環境として扱えます。

## ユースケース

Sprites を基盤にした SnapVM は、状態を保ったまま安全にコードを動かしたい場面に向いています。代表的な用途は次のとおりです。

- AI が生成したコードを隔離された環境で実行する
- ユーザー投稿コードや信頼できないコードを安全に試す
- セッションをまたいで状態が残る開発環境を提供する
- アイドル時に停止し、リクエストで復帰する HTTP サービスを動かす
- Git リポジトリや依存関係を含む CI/CD タスクを実行する

## Sprites のコア概念

### 永続性

Sprite はファイルシステムを保持します。実行中に書き込んだファイル、インストールしたパッケージ、チェックアウトしたリポジトリは、アイドル後の復帰時にも利用できます。

### 自動アイドル動作

使われていない Sprite はアイドル状態になり、必要に応じてハイバネーションします。次の接続やリクエストで復帰するため、常時稼働させずに状態を残せます。

### HTTP アクセス

Sprite 上で起動した HTTP サービスには URL 経由でアクセスできます。SnapVM ではマシンやサービスへのアクセスをプラットフォーム側の connect gateway 経由で扱います。
